## 2. 작업 세션 히스토리

### 세션 1 — 기본 기능 개발 및 최적화
- 결과 표시 버그 수정 (`st.rerun()` 후 분석 결과 사라지는 문제)
- `st.session_state` 기반 영속 저장으로 해결
- API 호출 최적화: yfinance 호출 4회 → 3회 (25% 절감)
- 30분봉 null 안전 처리
- 쿨타임 메커니즘 개선: `st.container()` 분리 렌더링
- 쿨타임 무한 루프 해결: 매초 rerun(10회) → 최종 1회 rerun으로 서버 부하 90% 절감

### 세션 2 — V1.0 정식 출시
- 쿨타임 UI 개선: 클라이언트 JS 카운트다운 (서버 리소스 0 소모)
- AI 이중 모델 폴백: Gemini 2.5 Pro(100회/일) → Flash(250회/일) 자동 전환
- 503 서버 과부하 에러 처리 추가
- 분석 히스토리 기능: 최근 10건 접기/펼치기 저장
- 분석 결과에 사용 엔진(Pro/Flash) 표기
- 프로젝트 브리핑 문서 작성

### 세션 3 — 프리마켓 거래량 문제 해결
- **문제:** yfinance가 프리마켓 시간대에 거래량을 0으로 반환
- **조사:** Alpaca, Twelve Data, Alpha Vantage, Polygon.io, Finnhub 등 무료 API 비교
- **Twelve Data 결과:** 프리마켓 거래량은 Pro 플랜($99/월)에서만 지원
- **최종 결정:** 비용 절감 위해 yfinance 유지, 거래량 0 감지 시 안내 메시지 추가
- `volume_unavailable` 세션 플래그 추가
- AI 프롬프트에 `vol_warning` 삽입 (가격 기반 분석 지시)
- 사용자에게 `st.warning()` 노란색 경고 표시

### 세션 4 — 미완성 봉 버그 수정 + 장 마감 안내 기능
- **문제1:** 정규장 중에도 프리마켓 경고가 뜨는 현상
- **원인:** `df.iloc[-1]`이 현재 진행 중인 봉(거래량 미집계=0)을 가져옴
- **수정:** 마지막 봉 거래량 0 + 직전 봉 거래량 > 0 → 직전 완성 봉 사용
- **결과:** 정규장에서는 완성 봉 사용, 프리마켓에서는 기존 경고 유지
- **추가 기능:** 장 마감 상태 감지 (EST 기준 주말/장외시간 판단)
- `is_market_open()` 함수로 정규장 09:30~16:00 EST 판단
- 장 마감 시 `st.info()` 파란색 안내 메시지 표시

## 3. 기능 명세서

### 3-1. 핵심 기능

| 기능 | 설명 |
|------|------|
| 멀티 타임프레임 분석 | 1분, 5분, 30분봉 동시 분석하여 단기/장기 교차 검증 |
| 거래량 정밀 판별 | 가격 변동이 진짜 거래량을 동반한 움직임인지 AI 분석 |
| AI 이중 모델 폴백 | Pro 소진 → Flash 자동 전환 (최대 350회/일) |
| 지능형 에러 핸들링 | 429/503/PerDay 에러별 차별화 대응 및 자동 재시도 |
| 글로벌 쿨타임 | 전역 10초 쿨타임, JS 카운트다운 (서버 부하 0) |
| 분석 히스토리 | 최근 10건 저장, 접기/펼치기 UI |
| 프리마켓 거래량 안내 | 거래량 0 감지 시 사용자 안내 + AI 가격 기반 분석 전환 |
| 미완성 봉 보정 | 현재 진행 중 봉 대신 직전 완성 봉 자동 선택 |
| 장 마감 안내 | 주말/장외시간 분석 시 마감 전 데이터 사용 안내 표시 |

### 3-2. 기술적 지표

분석에 사용되는 보조지표:
- SMA 5 (5이동평균)
- SMA 20 (20이동평균)
- 볼린저 밴드 (상단/하단)
- CCI (14기간)
- 스토캐스틱 K

### 3-3. 에러 처리 매트릭스

| 에러 | 대응 | 사용자 메시지 |
|------|------|--------------|
| 429 분당 제한 | 15초 대기 후 자동 재시도 (최대 2회) | ⏳ API 요청 한도 도달, 1분 후 재시도 |
| 503 서버 과부하 | 동일하게 15초 대기 재시도 | ⏳ 사용자 많아 대기 중 |
| PerDay 일일 소진 | 즉시 다음 모델로 폴백 전환 | 🚫 금일 무료 토큰 소진 |
| 데이터 없음 | 티커 유효성 안내 | 티커 확인 또는 장 중 확인 |

## 6. 주요 코드 변경 상세

### 미완성 봉 보정 로직 (2026-02-24)
**파일:** `stock_analyzer.py` → `get_stock_data()` 함수

```python
# 변경 전
return df.iloc[-1]

# 변경 후
if df['Volume'].iloc[-1] == 0 and len(df) >= 2 and df['Volume'].iloc[-2] > 0:
    return df.iloc[-2]  # 완성된 직전 봉
return df.iloc[-1]
```

**동작 원리:**
- 정규장 중: 현재 봉(거래량 미집계) → 직전 완성 봉 자동 선택
- 프리마켓/애프터마켓: 연속 거래량 0 → 기존대로 경고 메시지 표시

### 프리마켓 거래량 안내 (2026-02-24)
**파일:** `stock_analyzer.py`

```python
# 세션 초기화
if "volume_unavailable" not in st.session_state:
    st.session_state.volume_unavailable = False

# 거래량 0 감지
vol_warning = ""
if d1['Volume'] == 0 or d5['Volume'] == 0:
    vol_warning = "\n※ 프리마켓/애프터마켓 시간대로 거래량 데이터 미제공..."
    st.session_state.volume_unavailable = True
else:
    st.session_state.volume_unavailable = False

# 사용자 안내
if st.session_state.volume_unavailable:
    st.warning("📌 프리마켓/애프터마켓 시간대로 거래량 데이터가 제공되지 않습니다.")
```
